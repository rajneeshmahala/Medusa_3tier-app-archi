# ansible_vm_setup — Medusa v2 3-Tier Deployment

Deploys **Medusa v2** headless commerce across 3 Azure VMs provisioned by Terraform.

## Architecture

```
VM1 (Public)          VM2 (Private)         VM3 (Private)
─────────────         ──────────────         ──────────────
Next.js Storefront    Medusa v2 Backend      PostgreSQL 16
Medusa Admin (/app)   Node.js / Port 9000    Port 5432
Nginx :80             systemd service        systemd service

Browser → VM1:80 (Nginx) → VM2:9000 (Medusa API) → VM3:5432 (PostgreSQL)
```

## Access after deployment

| URL | What |
|-----|------|
| `http://<VM1_IP>` | Next.js Storefront |
| `http://<VM1_IP>/app` | Medusa Admin Dashboard |
| `http://<VM1_IP>/api/...` | Medusa REST API (proxied) |

**Admin login:** `admin@medusa-store.com` / `Supersecret1!`
(change in `group_vars/all.yml` before deploying)

## Before deploying — change secrets

Edit `group_vars/all.yml`:

```yaml
db_password:    "YourStrongDBPassword!"
jwt_secret:     "$(openssl rand -hex 32)"
cookie_secret:  "$(openssl rand -hex 32)"
admin_email:    "you@yourdomain.com"
admin_password: "YourAdminPassword!"
```

## Manual run

```bash
cd ansible_vm_setup/
ansible-playbook -i inventory.ini playbook.yml

# Single role only:
ansible-playbook -i inventory.ini playbook.yml --limit db
ansible-playbook -i inventory.ini playbook.yml --limit app
ansible-playbook -i inventory.ini playbook.yml --limit web

# Dry run:
ansible-playbook -i inventory.ini playbook.yml --check -v
```
